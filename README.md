## 这是什么

一个**自用**的 TikTok Shop 运营 Agent 产品，主攻 **马来西亚 · 跨境店（从中国发货）**。由三部分组成：

```
┌─ 驾驶舱 automation/ ─────────┐   ┌─ 引擎（任何 AI agent）────────┐
│ 本地网页：品卡看板 + 打分/定价   │   │ Claude Code / Codex / 人     │
│ 计算 + 人工决策 + 素材包导出    │   │ 按 AGENTS.md 协议干活：       │
│ python3 automation/run.py    │   │ 上1688找货源、马来语素材、答疑  │
└──────────────┬──────────────┘   └──────────────┬──────────────┘
               │        读写同一份文件              │
               └────────────┬───────────────────┘
                    ┌───────▼────────┐
                    │  workspace/     │ 数据总线：品卡 + 任务队列
                    │ （真实数据不入库）│ 协议见 workspace/README.md
                    └───────┬────────┘
                    ┌───────▼────────┐
                    │  reference/     │ 大脑：9 大板块运营知识
                    │ （+通用知识包）   │ 01开店…09风控，已核实
                    └────────────────┘
```

- **驾驶舱**做确定性的事：导入出海匠数据、打分、利润计算（RM）、看板与导出。
- **引擎**做需要脑子的事：找货源、翻译、写素材、回答运营问题。引擎不锁定模型——Claude 用浏览器 MCP，Codex 配 Playwright MCP，没有 AI 时人工手填照样能走完流程（三档：手动 / Codex / Claude）。
- **大脑**是唯一知识源：`reference/` 9 板块 + `通用知识包.md`（任何 AI 可直接消费）。

工程进度与里程碑见 **`ROADMAP.md`**（当前 v2 M0 已落地，M1 看板施工中）。

## 快速上手

**kk（技术）**：`python3 automation/run.py` 开驾驶舱 → Claude Code 里按 `AGENTS.md` 跑引擎任务。
**校长（运营）**：开驾驶舱看板做决策/导出上架 → AI 任务用 Codex（配置见 `AGENTS.md` §2）或手动填入。
**任何其他 AI**：读 `通用知识包.md` 当上下文即可回答运营问题；参与干活则按 `AGENTS.md`。

## 谁干什么（角色速查）

| 角色 | 谁 | 干什么 |
|---|---|---|
| 决策/审核/上架 | **校长（运营）** | 看板勾品、选货源、审素材、手动上架、盯测款 |
| 整合/自动化/基建 | **kk（技术）** | 审 PR 合并 main、最终整合知识、维护驾驶舱与协议 |
| 引擎/产出 | **AI（不限模型）** | 按 `AGENTS.md` 执行任务；按两遍消化制处理知识 |

> 知识贡献（喂料/两遍消化/PR 流程）见 `CONTRIBUTING.md` 第一部分；**代码贡献规范**（零依赖铁律/分层/DoD/CI）见第二部分。分工细节见 `sop/角色分工.md`；打法总纲见 `sop/TikTok-Shop-0到稳定出单-执行计划.md`。
