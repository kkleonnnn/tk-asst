# TikTok Shop 东南亚运营助理（知识库 + Claude Skill）

> **如果你是第一次打开这个仓库（无论你是人，还是某个 AI），请先读完本文件——它告诉你这是什么、你该干什么。**

## 这是什么
一个**自用**的 TikTok Shop 东南亚运营知识库。当前主攻 **马来西亚 · 跨境店（从中国发货）**。
它有两种"身份"：
- **对 Claude**：它是一个会被自动识别、自动触发的 **Skill**（入口 `SKILL.md`），还能配合浏览器实际操作店铺。
- **对任何其他 AI / 人**：`reference/` 目录和 `通用知识包.md` 是**纯文本知识**，可以直接读、直接用。

---

## 谁干什么（三种角色，请对号入座）

| 角色 | 谁/什么工具 | 能做什么 |
|---|---|---|
| **喂料** | 任何人、任何工具 | 把原始资料放进 `inbox/`（按格式命名） |
| **消化**（inbox → reference） | **只能 Claude 端 Opus 4.8 及以上**（无论用谁的账号） | 把原始资料提炼、结构化、核实后写进 `reference/` |
| **消费**（读 reference 来干活） | **任何 AI 模型** | 读 `reference/` 或 `通用知识包.md` 回答问题、起草 listing/话术/选品建议等 |

> ⚠️ **为什么消化只让 Opus 4.8+ 做**：消化要读 PDF/图片、交叉核实、按规范归类，质量直接决定知识库可信度。弱模型或其他 AI **不要直接改 `reference/`**——会污染知识库。它们只负责"消费"。

---

## 正式工作流程（一条龙）

```
①喂料            ②消化                 ③提 PR            ④合并 main         ⑤生成+消费
原始资料   →   Claude Opus4.8+   →   把 reference/   →   审核后并入 main →  重新生成
放进 inbox/     消化进 reference/      改动提 Pull Request   = 正式迭代生效     通用知识包.md，
（按格式）       草稿                  （不直接推 main）                       任何 AI 都能用
```

**关键：资料放进 inbox 后不会自动生效。** 必须经 ②Claude 消化 → ③PR → ④合并到 main，才算"正式迭代"进知识库。

## 给「非 Claude AI」的使用说明
如果你是 ChatGPT、Gemini 或其他 AI，被用来辅助 TikTok Shop 运营：
1. **你的知识来源 = `reference/` 里的内容（或整合好的 `通用知识包.md`）。** 优先以它为准，它是已核实的。
2. **不要把 `inbox/` 里未消化的原始资料当定论**——那些还没核实。
3. **不要直接改 `reference/`**——消化只允许 Claude Opus 4.8+ 做。你可以把内容当上下文来回答、起草、分析。
4. 你可以帮用户：写 listing/标题、客服话术、选品建议、大促方案、数据解读等——依据就是 reference 的知识。

## 给 Claude 的说明
见 `SKILL.md`。简言之：你被触发后用 `reference/` 的知识办事；**若被要求"消化 inbox"，仅当你是 Opus 4.8 及以上时才执行**，并把结果走 PR、合并 main。
