# TikTok Shop 东南亚运营助理（知识库 + Claude Skill）

> **如果你是第一次打开这个仓库（无论你是人，还是某个 AI），请先读完本文件——它告诉你这是什么、你该干什么。**

## 这是什么
一个**自用**的 TikTok Shop 东南亚运营知识库。当前主攻 **马来西亚 · 跨境店（从中国发货）**。
它有两种"身份"：
- **对 Claude**：它是一个会被自动识别、自动触发的 **Skill**（入口 `SKILL.md`），还能配合浏览器实际操作店铺。
- **对任何其他 AI / 人**：`reference/` 目录和 `通用知识包.md` 是**纯文本知识**，可以直接读、直接用。

---

## 谁干什么（三种角色，请对号入座）

| 角色 | 谁/什么工具 | 能做什么 |
|---|---|---|
| **喂料 + 第一遍消化** | 任何人、**任何模型** | 自备资料**本地先消化成 `.md`** 丢进 `inbox/`（按格式命名） |
| **最终整合**（inbox `.md` → reference） | **kk**（建议 Opus 4.8+ 把关） | 把 `.md` 校对/去重/核实/规范化并入 `reference/`、审 PR 合并 main |
| **消费**（读 reference 来干活） | **任何 AI 模型** | 读 `reference/` 或 `通用知识包.md` 回答问题、起草 listing/话术/选品建议等 |

> ⚠️ **两遍消化制**：第一遍人人可做（任何模型把资料消化成 `.md` 丢 `inbox/`），众包减负；**最终整合并入 `reference/` 是 kk 的质量关口**（校对/核实/规范化，建议 Opus 4.8+）。**其他人不直接改 `reference/`**，只交第一遍 `.md` 到 `inbox/`。
>
> 👥 **具体到人的分工**（谁运营、谁技术、谁经办注册）见 `sop/角色分工.md`；**整体打法与执行顺序**见旗舰执行计划 `sop/TikTok-Shop-0到稳定出单-执行计划.md`。

## 正式工作流程（一条龙）

```
①喂料 + 第一遍消化        ②最终整合(kk)            ③PR → 合并 main        ④生成 + 消费
自备资料本地消化成 .md  →  kk 把 inbox 的 .md     →  审核并入 main        →  重生成 通用知识包.md，
丢进 inbox/（任何模型）     校对/核实/规范化           = 正式迭代生效            任何 AI 都能用
                          并入 reference/
```

**关键：`.md` 放进 inbox 后不会自动生效。** 必须经 ②kk 最终整合并入 `reference/` → ③PR 合并 main，才算"正式迭代"进知识库。

## 给「非 Claude AI」的使用说明
如果你是 ChatGPT、Gemini 或其他 AI，被用来辅助 TikTok Shop 运营：
1. **你的知识来源 = `reference/` 里的内容（或整合好的 `通用知识包.md`）。** 优先以它为准，它是已核实的。
2. **不要把 `inbox/` 里未消化的原始资料当定论**——那些还没核实。
3. **你可以做"第一遍消化"**（把资料消化成 `.md` 放 `inbox/`），但**不要直接改 `reference/`**——并入 reference 是 kk 的最终整合。你也可以把内容当上下文来回答、起草、分析。
4. 你可以帮用户：写 listing/标题、客服话术、选品建议、大促方案、数据解读等——依据就是 reference 的知识。

## 给 Claude 的说明
见 `SKILL.md`。简言之：你被触发后用 `reference/` 的知识办事；**"消化 inbox"分两遍——任何模型可做第一遍（产出 `.md` 到 inbox）；并入 `reference/` 是 kk 的最终整合（建议 Opus 4.8+），走 PR 合并 main。**
