# 任务规范 · listing（上架素材）

> 引擎协议总则见根目录 `AGENTS.md` §2。本文件只讲 listing 任务怎么干。

## 目标
给任务里的每个品卡（已 `priced`：有选定货源和定价）生成**可直接用于手动上架的素材**，写进品卡 `listing{}`，状态推到 `listing_ready`。

## 输入
- 品卡 `source_data`（原商品名/类目/售价区间）、`sources[]` 里 `chosen_source_id` 对应的货源（含 1688 链接）、`pricing`（折后价/折前价 RM）。

## 产出内容（写进 `listing{}`，字段见 schema）
1. **`title_ms`** 马来语标题（≤255 字符）：套标题公式 = 核心词+关键词/属性+场景+人群+规格；用马来站搜索习惯用词；**禁绝对化词/未授权品牌词**。
2. **`title_en`** 英文标题（备用）。
3. **`selling_points_ms`** 3–5 条马来语卖点（对应主图/详情图文案）。
4. **`desc_ms`** 马来语详情描述：欢迎语 → 核心卖点 → 材质/规格 → 使用场景 → 尺寸/注意 → 物流售后（从中国发货 7–14 天口径，以后台实际为准）。
5. **`sku_names`** SKU 名（马来语，简洁，颜色/尺码等）。
6. **`compliance`**：对产出跑一遍合规——违禁词表与禁售类目见 `automation/core/compliance.py`（你可以直接运行它校验，或按同样规则人工过）；结果写 `{pass, issues[]}`。
7. **图片（尽力而为）**：从选定 1688 货源页收集主图/细节图，存到 `workspace/exports/<品卡id>/raw/`，路径写进 `images[]`。
   - ⚠️ **必查**：图里有中文/水印/联系方式/品牌 Logo 的，在 issues 里列明"第X张需处理"——**不做自动图片翻译**（做不好），处理交人工。
   - 拿不到图就留空并在 issues 说明。
8. `generated_by`（你的名字）、`generated_at`。

## 写回
- 品卡 `listing{}` 全量写入；`status` → `listing_ready`；`log[]` 追加。
- 任务 `done` + `result_note`（几个品完成、几个有 issues）。

## 红线
- 马来语要地道电商文风，不要机器直译腔；**不写疗效/绝对化/夸大宣称**。
- 清真/宗教敏感品类措辞谨慎（见 `reference/02-选品/`、`reference/09-风控合规/`）。
- 不编造材质/尺寸——1688 页面没写的参数，在 issues 标"待向供应商确认"。
