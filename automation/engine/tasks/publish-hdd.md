# 任务规范 · publish-hdd（货叮咚：采集→认领→收尾成草稿）

> 引擎协议总则见根目录 `AGENTS.md`。本文件把已实跑验证的货叮咚发布流程**锁死成固定步骤**，
> 目的：让引擎"照抄配方"而不是"边看边想"，把浏览器阶段的 token/时长压到最低。
> **红线：绝不点「保存并发布」/「认领并发布」——上架真商品不可逆，永远交人工。**

## 省 token 铁律（最重要，先读）
1. **值全部来自 recipe，不要自己想**。开工先取配方：`GET http://127.0.0.1:8765/api/publish-recipe?id=<PID>`。
   标题/描述/卖点/属性/重量/价格都在里面；SKU 采集后走 `POST /api/sku-translate`。想文案=烧 token 且不稳。
2. **别每点一步就截图**。截图是最大 token 黑洞。优先 `get_page_text`/`read_page` 一次读齐，`form_input` 批量填，
   `browser_batch` 合并多次点击。整个流程**只截 2 张图**：认领成功后、最终保存后（留证）。
3. **异步等待用一次 `wait` 再读**，不要连续截图轮询弹窗/新标签。
4. `recipe.warnings` 非空（非广东 / 缺文案 / SKU 有 unknown）→ **先停下回报，别硬发**。

## 前置
- 浏览器已登录货叮咚（`huodingdong.com`），目标店 = 马来店（如 Bleisure Vibe-MY）。
- 已有本品 recipe（见铁律 1）。`recipe.guangdong=false` 时按 warnings 停。

## 固定步骤
1. **链接采集**：开 `huodingdong.com/member/product/general/collect?tab=linkCollect`
   → 文本框粘 `recipe.source_url` → 点「**采集**」（不是「采集&认领」）→ 等"已提交采集任务"。
   （公开 1688 链接不装插件也能采。）
2. **认领**：开 `.../general/collect-box` → 「未认领」页 → 按 `货源ID` 找到该品 → 点「认领」。
3. **认领弹窗**（照选）：电商平台 `Tiktok`；地区 `马来西亚`；店铺 选马来店；
   **「认领并发布」开关保持 OFF（灰）**；认领配置模板用现有（系统默认或马来配置）→「确定」。生成草稿。**此处截图 1 张留证。**
4. **编辑草稿**：`产品发布 → 草稿箱` → 该品「编辑」（新标签 `tiktok/edit?...&scene=draft`）。
5. **基本信息**：
   - 产品标题：点入 → 全选(Cmd/Ctrl+A) → 输入 `recipe.title_ms`。
   - 图文描述：点入编辑器 → 全选 → 删除（连带清掉中文详情图）→ 输入 `recipe.desc_ms`（可接 `selling_points_ms`）。
6. **销售信息（SKU 名）**：`get_page_text` 读出 SKU 列表的中文名（保持顺序）
   → `POST /api/sku-translate {"names":[...]}` 拿 `ms` → 逐个字段全选替换（用 `browser_batch` 合并，别逐格截图）。
7. **类目&属性**：类目已自动匹配则不动。按 `recipe.attributes` 选：品牌=`无品牌`（或 `(Trade mark without text)`）、
   材质=`塑料材料`、功能=`可折叠的`、形状=`长方形`、存储类型=`带盖子`（均为下拉/多选）。
8. **物流**：包裹重量若为空，填 `recipe.weight_g`；已填（认领带入）则保留，改到 warnings 里提示人工核。
9. **保存**：点「**保存修改**」（存草稿）。**最终截图 1 张留证**。→ 状态回报：草稿已就绪，待人工（配促销折扣 + 洗主图中文 + 发布）。

## 写回（品卡）
- 成功 → 品卡 `log[]` 追加「货叮咚草稿已建（店/草稿标题）」；不改价格/状态机以外字段。
- 失败/中断 → 照实写明卡在哪一步，`result_note` 说明，别谎报成功。

## 红线
- 「认领并发布」OFF；「保存并发布」绝不点。
- 不写账号/密码/验证码进任何文件。
- 主图去中文水印/图片翻译是**付费/人工**环节，不在本任务内替人做；在回报里提示交校长。
