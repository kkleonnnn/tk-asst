# 怎么给这个助理"喂资料" / 迭代它

> 给不写代码的伙伴：这个 skill 就是一堆文本文件，**改它 = 改文字，不需要写代码。**

## 最省事的方式：只管投喂，不管整理
1. 把新学到的东西（截图、文档、聊天记录、自己的笔记）直接丢进 `inbox/` 文件夹。
2. 文件名随便起，能看懂就行，比如 `2026-06-21 马来西亚发货流程截图.md`。
3. 之后由 Claude 或队友把它"消化"进正式的 `reference/` `sop/` 里。

**投喂时只要做到 3 点：**
- ✅ 写清楚**来源和日期**（哪看来的、什么时候，TikTok 规则常变）
- ✅ 一条资料尽量只讲**一件事**
- 🚫 **绝不**写账号密码、API key、买家隐私

## 进阶方式：让 Claude 帮你改
直接对 Claude 说：「我学到 XX，把这条加进运营 SOP」。Claude 会帮你改好文件，你在 GitHub 网页点一下提交即可。

## 直接改（可选）
- **GitHub 网页**：打开文件 → 点铅笔图标编辑 → 底部 Commit。
- **GitHub Desktop**：桌面应用，改完点 Commit、点 Push，全程不碰命令行。

## 目录速查
| 文件夹 | 放什么 |
|---|---|
| `inbox/` | 新资料先丢这里 |
| `reference/` | 整理好的知识（分主题） |
| `sop/` | 一步步照着做的操作流程 |
| `templates/` | 文案 / 报告 / 话术模板 |
