## 〇、入口（三大工具公共路径）

**卖家后台 → 左侧「广告营销」→ 「促销活动」**

进去后可见平台支持的各类促销工具，最常用三个：

1. **商品折扣**（百分比 / 一口价打折）
2. **商家秒杀**（限时秒杀，3 天有效期）
3. **优惠券**（聊天券 / 评价券 / 满减券等）

> ⚠️ 入口文案"广告营销""促销活动"以当期后台为准；不同站点 / 版本可能叫"营销中心""促销工具"。

---

## 一、三件工具的核心规则（先记规则，再动手）

> **这一节是最容易踩坑、最该背下来的部分。** 数字均为讲稿口径，以官方后台规则页为准。

### 1. 叠加 / 互斥关系（决定买家最终看到什么价）

| 组合 | 生效规则 |
|---|---|
| 同一商品参加**多个商品折扣** | 任意时间段内**只生效折扣最大（最便宜）的那个**，只显示最优惠价 |
| **商品折扣 + 商家秒杀** 同时存在 | **不叠加**，优先显示**秒杀价**（秒杀价覆盖折扣价） |
| **优惠券 + 商品折扣 / 商家秒杀** | **可叠加**（券在折后/秒杀价基础上再减）|
| 同一订单的**商家优惠券** | 一个订单**最多只能用一张**商家券 |

→ 实务含义：想让买家叠到"折扣 + 券"或"秒杀 + 券"是允许的；但别指望"折扣 + 秒杀"同时减，平台只认秒杀价。

### 2. 商家秒杀的硬性限制

- **有效期只有 3 天**：到期即结束，想继续做要**重新创建一遍**秒杀活动。
- **秒杀价必须低于该商品过去 14 天内的最低成交价**；
- 若该商品过去 14 天**无成交记录**，则秒杀价需**低于原价**即可。

### 3. 「指定销售变体」是防亏损红线（折扣 / 秒杀通用）

设置商品时有两个选项：**指定商品** vs **指定销售变体**。

- **务必选「指定销售变体」**：按每个 SKU 各自价格分别算促销价，不同 SKU 展示不同促销价。
- **若选「指定商品」而商品下多个 SKU 价格不同** → 系统会**按 SKU 里的最低价去套促销**，高价 SKU 也被打到低价基准上，**直接造成亏损**。
- 记忆点：**只要一个商品有多个不同价 SKU，就一定选「指定销售变体」。**

## 二、商品折扣 设置 SOP

路径：促销活动 → **商品折扣 → 创建**

1. **活动名称**：自定义或用系统默认均可。
2. **活动时间**：用系统默认或自定义起止时间。
3. **折扣类型**二选一：
   - **百分比折扣**（最常用）——按比例打折；
   - **一口价**——固定促销价（设置时注意别填错，固定价直接覆盖）。
4. **商品范围**：选 **「指定销售变体」**（见上 §1.3 防亏损）。
5. 勾选要做折扣的商品 → **完成**。
6. 在折扣输入位**打勾**，填入**折扣百分比**。
   - **折扣力度 = 跟着你的定价模板走**（关键换算，见下）。
7. 点右侧 **「批量更新」** → 下方自动算出**促销价**。
8. 核对促销价无误 → 右上角 **「同意并发布」**。
9. 发布后可在 **「未开始」** 标签页看到该活动。

### 折扣百分比怎么填（对应定价模板的换算）

讲稿给的是"百分比 = 让利掉的那部分"的口径（即输入框填的是**降价幅度**，不是最终价折扣率）：

| 定价模板设定的目标折扣 | 在折扣框填入的百分比 |
|---|---|
| 打 **7 折**（卖到原价 70%） | **30%** |
| 打 **6 折**（卖到原价 60%） | **40%** |

> 规律：**填的百分比 = (1 − 目标折扣率) × 100%**。
> ⚠️ 这是讲稿口径，不同后台版本对"百分比折扣"输入框的语义可能不同（有的填的是"最终价占原价的比例"）。**第一次设置务必先填一个、点批量更新、看下方算出来的促销价对不对，再批量套用。**

## 三、商家秒杀 设置 SOP

路径：促销活动（促销工具）→ **商家秒杀 → 创建**

1. **活动名称**：自定义或默认。
2. **展示渠道**三选一：
   - **全部**（一般选这个，无局限性）
   - 仅限直播
   - 仅限达人直播
   - → 按店铺运营方式选；纯货架/无直播就选全部。
3. **开始 / 结束时间**：注意**秒杀有效期仅 3 天**（见 §1.2）。
4. **商品范围**：选 **「指定销售变体」**（多 SKU 不同价时尤其重要，否则按最低价套秒杀亏损）。
5. 勾选要秒杀的商品 → **完成**。
6. 打勾 → 输入**秒杀折扣价**（注意须满足"低于 14 天最低价 / 低于原价"，见 §1.2）。
7. **批量更新** → 出现秒杀优惠价。
8. 核对无误 → **同意并发布** → 提示"促销创建成功"。
9. 在 **「未开始」** 可查看。

> **续期提醒**：3 天到期后秒杀自动结束，要继续得**重新创建**。大促期若需连续秒杀，提前排好"每 3 天重建一次"的节奏。

## 四、优惠券 设置 SOP

路径：促销活动 → **优惠券 → 创建**

### 先选对券类型（决定谁能看到 / 领到）

| 券类型 | 特性 / 用途 |
|---|---|
| **聊天优惠券** | **只能通过聊天消息单独发给某个客户的专属券**，其他买家看不到。→ **想发"专属券 / 一对一挽单券"必须用这个**。|
| **评价优惠券** | 平台较新推出；**鼓励买家多评价**，买家评价符合平台标准后**自动发放**。|
| 其它公开券（如满减券） | 设了之后**所有买家都能看到 / 领取**。|

> 记忆点：**专属券 = 聊天券**；其它券都是公开的。

### 设置步骤

1. **优惠券名称**：自定义或默认。
2. **开始 / 结束时间**。
3. **展示渠道**：全部渠道 / 仅商城 —— 一般选**全部渠道**。
4. **优惠券有效期**：设买家领取后多久内可用。
5. **折扣力度**二选一：
   - **固定金额折扣**（最常用）——固定面额，例：15 THB 券面额即固定减 15 泰铢（**MY 站换成 RM**）；
   - 百分比折扣。
6. **消费门槛**：
   - **具体金额门槛**：买家消费达到该金额才能用此券；
   - 或 **无门槛**（无订单金额限制）。
7. **单个客户可领数量** + 提醒：**每笔订单最多用 1 张商家券**（见 §1.1）。
8. **适用范围**：所有商品 / 特定商品。
9. 核对无误 → 右上角 **「同意并发布」**。
10. 在 **「未开始」** 可看到已设的券（与秒杀、折扣并列展示）。

## 五、活动管理：终止 / 查看

- **查看待开始活动**：促销活动列表 → **「未开始」** 标签页（折扣 / 秒杀 / 券都在此）。
- **提前终止进行中的活动**：
  1. 切到 **「进行中」** 标签页；
  2. 找到要终止的活动 → 点右侧 **「⋮」（三个点）**；
  3. 选**终止**即可。

## 六、避坑速记 + MY 跨境店落地提醒

**避坑（操作层）**
1. **多 SKU 不同价 → 永远选「指定销售变体」**，否则按最低价套促销，高价 SKU 亏本。
2. **折扣百分比先点"批量更新"看算出的促销价再发布**，别凭记忆直接套（避免把折扣方向填反）。
3. **折扣 ≠ 叠加秒杀**：同品别又设折扣又设秒杀以为更便宜，平台只显示秒杀价。
4. **秒杀只活 3 天**：要连做得排重建节奏。
5. **秒杀价别忘了"低于 14 天最低价"**这条，价不够低可能创建失败 / 不合规。
6. **想发专属/挽单券 → 只能用聊天券**，其它券所有人可见。

**MY 跨境店落地**
- 讲稿面额举例是**泰铢**，本店一律换成 **RM（MYR）**；优惠券面额、满减门槛要按 **RM 价格带**重设并测试（参考 `大促作战.md` "营销玩法"里"泰铢示例需换成 RM 阶梯"的同款提醒）。
- 这三件工具**平台免费**，合理组合（折扣养价 + 秒杀冲爆 + 券拉转化/挽单）可提流量与转化；具体力度、节奏跟 **`大促作战.md`** 的预热/爆发/沉淀三段式配合使用。
- 折扣力度回到**定价模板**算（见 §二换算表），别脱离成本拍脑袋打折。

---

> **一句话**：入口是「广告营销 → 促销活动」；三件套规则=折扣取最优、秒杀盖折扣且仅 3 天且须破 14 天最低价、券可叠加但一单一张；所有商品范围都选「指定销售变体」防亏损；面额泰铢示例 MY 换 RM；规则数字以当期后台为准。
