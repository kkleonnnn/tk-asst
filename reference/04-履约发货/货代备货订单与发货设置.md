## 0. 一页速记

| 项 | 要点 | 可信度 |
|---|---|---|
| 工具 | 货叮咚 ERP（第三方，非官方）+ 浏览器采购插件 | 高 |
| 核心链路 | 授权店铺 → 授权货源(1688) → 授权货代 → 订单设置 → 订单管理 → 插件/API 采购发货 或 库存发货 → 拒收/退件 | 高 |
| 发货地址 **必须先填** | 不填**发货地址+退货地址**，货叮咚后台**无法上架发布商品** | 高 |
| **揽收范围外**地址 → 发货时效 **96h（4天）** | 填揽收范围**外**地址，时效宽松（推荐） | 高（与官方政策挂钩）|
| **揽收范围内**地址 → 发货时效 **48h（2天）** | 页面出现「申请上门取件」=在揽收范围内；时效更紧 | 高 |
| 退货地址（国内） | **建议填货代仓库地址**，便于货代代收退货并返还货源商家 | 中（业务建议）|
| 本地退货方式 | 无货源商家**建议选「退到 TK 官方认证仓」** | 中（业务建议）|
| 多站点 | 每个站点都要单独设地址；改一个仓库地址会**联动同仓所有站点** | 高（避坑）|
| 货代角色 | 负责**打包+发货+物流**；无货源代发**必须授权一个货代仓库**才能开启待打包 | 高 |
| 出货模式（订单设置） | 无货源模式 / 部分囤货模式 / 库存订单模式，三选一适配卖家类型 | 高 |

> **马来专属 vs 通用**：揽收时效 48h/96h、TK 官方认证仓退货 → 与马来/东南亚官方政策强相关；货叮咚的授权/采购/库存/拒收操作 → 跨平台通用，不限马来。

## 1. 总流程：货叮咚 ERP 十步（订单篇主线）

> 来源：`TikTok新手教程（订单篇）_原文.docx`。这是把零散教程串起来的主线，**新手按此顺序走一遍**。

1. **授权店铺** —— 让货叮咚拿到订单同步权限。
2. **货源授权** —— 授权货源账号（如 1688），解锁采购自动派单、直连官方 API。
3. **货代授权** —— 无货源代发**必须**授权货代仓库，开启待打包服务（详见 §2）。
4. **订单设置** —— 选出货模式 + 打包窗口设置（详见 §3）。
5. **订单管理** —— 日常主战场，看订单状态流转与字段（详见 §4）。
6. **插件安装** —— 装浏览器采购插件（详见 §5.1）。
7. **订单采购发货** —— 插件采购 或 官方 API 采购（详见 §5）。
8. **订单库存发货** —— 已备货配对的商品走库存出货（详见 §5.3、§6）。
9. **货代常见问题** —— 仓库地址/采购地址/问题包裹/异常订单（详见 §7）。
10. **拒收退件管理** —— 拒收单 / 退件单（详见 §8）。

### 1.1 授权 SOP（步骤 1–3 细节）

- **TikTok 店铺授权**：系统 → TikTok 店铺授权 → 店铺授权 → 输入站点账号密码登录授权 → 选授权地区/站点 → 输入邮箱确认安装 → 确认授权 → 成功。
- **1688 货源授权**：货源账号 → 1688 授权 → 立即订购 → 输入手机号、勾协议 → 提交订单 → 勾选并授权 → 成功。（直连官方 API 采购派单，一站式同步采购。）
- **货代授权**（无货源代发必做）：
  1. 订单 → 货代管理 → 首次使用点「注册」。
  2. **货代商家端是单独注册/管理的账号**：注册为货代商家端 → 登录商家端 → 商家端可单独做「店铺授权」和「打包订单」，**需先授权店铺才能推送货代订单**。
  3. 回货叮咚 → 「授权货代」→ 获取授权码 → 复制商家端授权码 → 粘贴 → 确认授权 → 成功。
  4. 「我的货代服务商」里可对已授权货代**充值**等操作。

## 2. 货代授权与默认打包仓

- **货代是负责打包+发货+物流的关键环节**；无货源代发商家**必须授权一个货代仓库**才能开启待打包服务。
- **设默认打包仓**（来自 `货代常见问题_原文.docx`）：
  - 货代端首页可查看**支持的仓库和仓库地址**。
  - 商家端可把对应仓库**设为货代默认打包仓库**；也可在**打包页面选中发货仓库 → 设为默认仓库**，设好后能看到「默认仓库标识」。
- **充值入口**：货代服务商页面提供快捷充值和系统入口。

> ⚠️ 货代仓库地址要和**店铺后台发货地址**保持一致（见 §9 衔接）——出单后货代按面单地址把货送进 TK 官方仓。

## 3. 订单设置（出货模式 + 打包窗口）

> 来源：`订单设置教程_原文.docx` ＝ `订单篇` 的子集，已合并去重。
> 路径：货叮咚 → 订单 → 订单设置（决定订单页快捷内容与出货方式）。

### 3.1 三种出货展示模式（按卖家类型三选一）

| 模式 | 优先展示 | 适用卖家 |
|---|---|---|
| **优先无货源模式** | 货源 | **无货源代发为主**（本店若纯代发选此）|
| **部分囤货模式** | 库存 | 备货为主、兼采购代发 |
| **库存订单模式** | 仅库存 | 仓库现货 / 海外仓现货 |

> 同一组模式在「订单设置」和「订单管理 → 订单信息」两处都能切，保存后刷新生效。

### 3.2 打包窗口设置（逐项）

- **货代备注标签**：自动预报按钮、**预报手机号一一对应**（打包页默认快捷填写）。
- **默认货币**：按使用习惯切换默认货币展示。
- **拣货单设置**：默认拣货单打印模板。
- **移入待打单规则**：可选「**出货后流转**」或「**打包后流转**」。
- **打包状态商品默认展示语言**：支持切换中文（更易读）。
- 保存配置后**刷新页面生效**。

## 4. 订单管理（状态流转 + 订单字段）

> 来源：`订单管理教程_原文.docx` ＝ `订单篇` 子集，已合并。日常运营大部分时间在此页。
> 路径：货叮咚 → 订单。

### 4.1 订单状态流转（无货源代发链路，重点记顺序）

| 状态 | 关注点 |
|---|---|
| 待处理（**未申请运单号**） | 采购拍单、待打单发货 |
| 已申请运单号 | 仓库出货；未出货时**区分采购订单是否已发货**，注意**发货时间** |
| 已出货 | 采购订单发货后状态流转，注意**交运时间** |
| 交运中 | 货代完成打包出货 |
| 已发货 | 货物进入物流运输环节 |

- **筛选/搜索**：可按店铺、仓库过滤；支持输入**订单号 / 快递单号**精准搜索。
- **同步**：系统实时自动同步订单，也可**手动导出/同步**。

### 4.2 订单信息字段（查单时能看到什么）

- **基础**：下单店铺、当前订单状态、平台买家账号、买家留言、商品信息（标题/图片/规格/价格）。
- **账单**（可切人民币或对应货币）：下单金额、运费金额、平台扣点、其他费用明细。
- **物流**：买家收货地址、物流信息、**平台物流运输单号**。
- **时间**：下单时间、最后发货时间、仓库出库时间、最后订单商品对应的出货展示。

## 5. 采购发货（插件 / API）

### 5.1 安装采购插件（步骤）

- 货叮咚支持 **1688 / 淘宝 / 拼多多** 插件拍单，在线自动同步发货单号。
- 订单页上方可**检测派单插件是否安装、派单账号是否在线**。
- 安装（以 Chrome 为例）：下载插件 → 浏览器右上角 → 扩展程序 → 管理扩展程序 → **开启「开发者模式」** → 把下载的货叮咚插件**拖入安装** → 完成。

### 5.2 插件采购发货 SOP

1. **先绑货源**：用插件采购前，对未绑定货源的商品**关联货源**（支持手动加电商平台货源链接、「快捷搜同款」关联）；绑定后显示货源信息。
2. 批量选中待采购订单 → 选「批量采购拍单」→ 核对拍单仓库信息 → 发起采购。
3. **首次使用先设「智能派单」**：自动匹配 SKU、自动匹配数量、跳过支付利润预警、**开启自动支付**后确认保存。
4. **首次下单手动**选排单 SKU，核对商品利润和地址信息 → 立即支付（系统自动支付，提高派单效率）→ 支付完成自动跳下一个采购订单。
5. 商家发货后**定时同步发货单号、自动提交仓库打包**；所有订单采购完成后自动绑定采购订单号、采购成本，自动提交仓库打包。

### 5.3 官方 API 采购（比插件更快更稳）

- 支持**淘宝、1688**；教程以 1688 批量采购演示。
- 批量选采购订单 → 选「批量 1688 采购」→ 未绑货源可「快捷搜同款」绑定 → 一键移除未绑货源商品 → 不满足批发条件的可**切换代发下单**。
- 首次下单选下单规格（后续自动默认匹配）→ 预览采购单 → **二次核对**：单笔提交金额、运费、商品利润、预估总利润、总采购数量、仓库地址、下单账号 → 提交订单（可稍后登录支付，或直接提交并支付，跳 1688 批量付款）。
- 付款完成自动绑定订单号、同步采购成本；商家发货后自动同步发货单号、提交仓库打包。

### 5.4 出货收尾

- 采购完成或库存打包完成的订单**流转到「待打单发货」**状态。
- 仓库接收出货订单后进行**拣货 → 出货交运**。

## 6. 备货：商品管理 + 入库管理（囤货卖家）

> 来源：`备货商品管理_原文.docx`、`备货入库管理_原文.docx`。适用于走「部分囤货/库存订单」模式的卖家。
> 路径：货叮咚 → 库存（即「货叮咚备货 / 货叮咚库存管理」）。

### 6.1 备货商品创建（3 种方式）

| 方式 | 操作 | 适用 |
|---|---|---|
| **① 在线商品快捷添加** | 一键同步商品 SKU，自动填入名称/规格/价格，自动关联店铺商品，保存即建好 | 已在店铺在线的商品（最快）|
| **② 手动添加** | 手动录入商品详情；SKU 参数**可直接引用店铺在线商品**，无需重复关联，保存即建好 | 线下/未在线商品 |
| **③ 组合商品** | 把多个单库存商品整合成一个**套装**：关联添加一个商品信息 → 添加多个库存商品作为关联单品库存 → 与店铺规格关联（可设赠品套装）→ 保存 | 套装/赠品组合 |

### 6.2 商品配对（备货商品 ↔ 店铺商品）

- 目的：把**备货商品**与**店铺商品 SKU**关联起来，出单后库存打包能自动匹配。
- **一键配对**：按预设映射条件，系统自动完成备货商品与店铺商品 SKU 匹配。
- **手动配对**：手动绑定备货商品与店铺商品规格，灵活适配不同场景。
- 效果：**订单出单后，库存打包自动提示关联的库存商品，一键提交发货**，出货效率翻倍。

### 6.3 入库管理（入库 → 出库全程可控）

路径：库存 → 入库管理。

1. **入库申请**：选入库仓库 → 填**发货单号 / 发货到仓库的物流单号**（可加多个 + 备注）→ 添加要入库的商品（可批量选）→ 填**到货数量**（方便仓库核对清点）。
2. 快递到货后，仓库**确认数量、规格无误 → 完成入库上架**，记录操作，可快捷查看**入库日志**。

### 6.4 库存看板（备货决策）

- **现有库存**：查看备货库存，规避缺货断单与滞销积压。
- **库存预警**：结合**近期销量 + 库存数量**判断备货是否充足，避免断货，提醒补货周期。
- **货值**：查看总库存货物投入价值，规避库存积压的资金占用风险。
- **库存日志**：每个库存订单的详细出入库数据，掌握每款商品的**累计入库/累计出库**记录，合理规划备货数量。

### 6.5 库存发货 SOP

- 路径：订单 → 库存发货（**需先按 §6 创建商品并入库上架**）。
- 部分已备货配对的商品 → 点「库存发货」→ 选对应仓库和条件 → 选库存商品 → 提交库存发货。
- 也可在**打包出货页**编辑：选打包仓库 → 选库存发货 → 选备货商品 → 提交。
- 完成后订单流转「待打单发货」，仓库拣货出货交运。

## 7. 货代常见问题（卡点处理）

> 来源：`货代常见问题_原文.docx`（与 `订单篇` 末段一致）。新手最常卡：仓库地址、采购地址、包裹异常。

- **支持仓库 / 仓库地址**：货代端首页可查看；商家端设默认打包仓见 §2。
- **问题包裹**：当**采购发货但未关联打包订单的仓库**出现时，系统通过「问题包裹」通知你**及时录单提交打包**；可查看签收单号对应的仓库信息和包裹图片。
- **问题订单（货不对单）**：仓库核验**货物与订单产品不一致**时，需确认是否发货；系统会发消息说明 →
  1. 点对应订单号**跳转处理**；
  2. 打开消息窗口看异常消息和图片确认；
  3. 在聊天页**回复处理方式、标记已读** → 异常订单消息变更为「已处理」。

## 8. 拒收 / 退件管理

> 来源：`订单篇` 第十部分。买家拒收、包裹退回、售后退换货时在货叮咚里处理。
> 路径：订单 → 拒收管理 / 退件管理。

### 8.1 拒收（**在途、未签收**）

- 场景：在途运输包裹，因客户取消订单 / 多拍 / 错拍等，**签收前**直接操作拒收 → **节省退件成本**。
- SOP：点「新增拒收单号」→ 选包裹收货仓库 → 录入**发货快递单号** → 拒收成功后包裹状态变「**已拒收**」。

### 8.2 退件（**已签收**）

- 场景：已签收包裹，客户取消 / 多拍 / 错拍 / 商家错发等。
- SOP：点「新增退件单号」→ 选包裹签收仓库 → 录入发货快递单号 → **填写指定退货地址**（上门取件可备注**取件码**）→ 退件成功后状态变「**已退件**」，可查看退货物流公司与退件单号。
- 方式：**上门取件** 或 **仓库退件**，退回到指定退货地址。

### 8.3 收尾

- 拒收/退件成功后，**留意采购平台（1688 等）的退货退款**——把货代侧退回和上游采购退款对上，避免损失。

## 9. 店铺后台「发货地址 / 退货地址」设置（重点 SOP）

> 来源：`TK小店发货地址设置教程_原文.docx`（+ 同名 PPT）。**这是官方卖家后台的设置**，不是货叮咚里的——但与货代/履约强绑定。马来场景。

### 9.1 为什么必须先设

- **不设发货地址+退货地址，货叮咚后台无法上架发布商品**。操作前务必先填。
- 设地址的意义：让**货代打印对应面单**——主要 3 种面单：**洪梅面单、万盛面单、顺丰面单**（讲稿语音，疑为「洪梅 / 万色 / 顺丰」，与批次1分拨中心对应）。出单后货代按面单地址把货送进对应 TK 官方仓。
- ⚠️ 面单名称为讲稿口述，**以货代/后台实际提供为准**。

### 9.2 设置 SOP（发货地址）

1. 登录店铺后台 → **首页**有「设置发货地址和退货地址」入口；或右上角 → **我的账号 → 商家资料 → 仓库取件地址**（这里能看是否已设，空白＝未设）。
2. 首页 → 点「去设置」→ 跳到填写页。
3. 填**仓库名称、联系人、手机号码**。
   - **还没找货代之前**：先填你当前的**居住地址**即可；**找到货代后再改成货代仓库地址**。
4. 填**邮政编码**：复制你填的地址 → 在浏览器搜该区域邮编 → 复制粘贴。（例：地址在浙江义乌稠江街道，就搜该街道邮编；地址在东莞就搜东莞对应邮编。）

### 9.3 退货地址（同页往下）

1. **国内退货仓库地址**：可直接选「取件仓库地址（第一个）」。
2. **本地退货方式**（3 选项）：**无货源商家建议选第二个——「退到 TK 官方认证仓」**。下面的地址按系统默认即可。
   - 说明：买家退款时平台会**按买家所在地就近**变动，退到离买家最近的仓库地址，按系统默认勾选即可。
3. 勾选「我已阅读并同意上述内容」→ 检查无误 → 右上角「提交」→ 显示「设置成功」即完成。

### 9.4 国内退货地址为何建议填货代仓库

- 场景：订单在**出境前**被取消/未出库，货代已把货送到 TK 官方仓 → 平台会把货退到**店铺后台设置的退货地址**。
- 所以**国内退货地址建议填货代仓库地址**，让货代代收后**返还货物给货源商家**。
- 香港仓：右上角「添加仓库」可手动加**中国香港仓库地址**（有的话）。

### 9.5 ⭐ 揽收范围内/外 → 直接决定发货时效（最关键避坑）

| 地址类型 | 如何判断 | 发货时效 |
|---|---|---|
| **揽收范围外** | 页面**没有**「申请上门取件」字样 | **96 小时（4 天）** ← 推荐，更宽松 |
| **揽收范围内** | 仓库发货地址右边出现「**申请上门取件**」6 个字 | **48 小时（2 天）** ← 更紧 |

- 怎么确认自己填的是不是范围外：右上角「**帮助和支持 → 学习中心**」搜索；或看店铺仓库发货地址右边有没有「申请上门取件」。
- **正常情况不要点「申请上门取件」**：一点就进入揽收范围内、时效收紧到 48h，且 TK 官方仓会派人上门收件。
- **接到 TK 官方确认上门揽收的电话**（打的是你后台填的发货地址电话）→ **直接答「不需要」**，平台就不会上门，订单发货时效保持 **96h（4 天）**。

### 9.6 多站点设置（避坑）

- 做多站点的，要**逐个站点**切换后分别设置发货地址 + 退货地址（右上角切站点）。
- 未设的站点，「仓库取件地址」页会显示**空白**，需回首页「去设置」补。
- ⚠️ **改一个仓库地址会联动同一仓库下所有站点**——不能只改单一站点；改完点右下角「保存」。
- 切到其他站点时，若发货地址已是同一货代仓，通常只需补设**当地退货地址**（选 TK 官方认证仓 → 勾选 → 提交）。

## 10. 与既有规则的衔接 / 待核实 / 冲突标注

- **时效口径对齐（与批次1一致，可信度高）**：本文 §0/§9 的「揽收范围内 48h、范围外 96h」与批次1 `发货指南与运费.md` 的「付款后 48h 打包发运、自寄 96h 到仓」是**同一套官方时效红线的两种表述**，互相印证。但 README 里写「跨境自寄已从 120h 收紧到 72h」——**这与本文/批次1 的 96h 不一致**，疑为更新口径或不同产品/场景，⚠️ **以官方后台学习中心当期为准**，下单前务必核实到底是 72h 还是 96h。
- **退货地址逻辑**：本文「国内退货地址建议填货代仓 + 本地退货选 TK 官方认证仓」与批次1「中国大陆分拨仓退货 + 东南亚当地退货地址」描述的是同一机制的卖家侧操作，互补。
- **货叮咚 是第三方 ERP，非官方**：所有菜单/按钮/「智能派单/自动支付」等以软件当期界面为准；是否启用、充值、选哪家货代属商业决策，本文不构成推荐。
- **面单名称**（洪梅/万盛/顺丰）为讲稿口述，可能是「洪梅/万色/顺丰」笔误，**以货代实际提供面单为准**。
