## 0. 全流程总览（8 步 + 一个事后回填）

货叮咚新手教程把"从采集到上架"拆成 **8 个核心步骤**，本文按此主线 + 事后成本回填：

| # | 步骤 | 在货叮咚的位置 | 本文章节 |
|---|---|---|---|
| 1 | 授权店铺（TikTok 店铺授权） | 系统 → TikTok 店铺授权 | §1.1 |
| 2 | 插件安装 | 产品 → 下载货叮咚插件 | §1.2 |
| 3 | 采集配置 | 产品 → 采集配置 | §2.1 |
| 4 | 商品采集 | 产品 → 插件采集 / 1688精选货源 | §2.2 |
| 5 | 认领配置 | 产品 → 认领配置 | §3 |
| 6 | 发布配置 | 产品 → 产品发布（发布配置） | §5.1 |
| 7 | 产品编辑 | 采集箱/草稿商品 → 编辑 | §5.2 |
| 8 | 产品发布 | 草稿商品 → 发布/定时/批量发布 | §5.3 |
| 事后 | **采购派单 + 利润明细回填成本** | 订单（采购）；数据 → 利润明细 | §6、§7 |

> 整体心智模型：**采集箱**（采来的原始货）→ 认领（按"认领配置"标准化、套定价模板、生成**草稿商品**）→ 发布（按"发布配置"上线到 TikTok）→ 出单后**采购派单**到 1688 → 最后在**利润明细**把采购价/货代费回填，利润才算准。

## 1. 环境准备：授权 + 装插件

### 1.1 TikTok 店铺授权（第 1 步，全功能前提）
- 位置：`货叮咚 系统 → TikTok 店铺授权 → 店铺授权`。
- 操作：输入**站点账号和密码**登录 → **选择授权地区站点** → 输入邮箱确认安装 → 确认授权 → 授权成功。
- 作用：让货叮咚 ERP 拿到你 TikTok 店铺的操作权限。**铺货、营销管理、采购派单都需要先授权店铺。**

### 1.2 货叮咚采集插件安装（第 2 步，不装无法采集）
> ↔批1 §2.1 已提过"插件拖进扩展程序"。增量：**支持 3 种浏览器**、各自路径不同、必须开开发者模式、自检方法、升级提示。

- 下载入口：登录货叮咚 → `产品 → 插件采集`（页面下方列出适配的浏览器及对应安装包）。常用 3 个浏览器：
  - **谷歌 Chrome**（首选）
  - **Edge**（用 Chrome 的同一个安装包）
  - **360 浏览器**（⚠️教程明确**不建议**：360 下载"可能会有很多病毒"）
- **Chrome 安装步骤**：右键安装包 → 下载 → 右上角"⋮" → 扩展程序 → 管理扩展程序 → **必须打开「开发者模式」（否则装不上）** → 右上角下载按钮找到刚下的安装包 → 点文件夹图标打开所在目录 → **把安装包文件拖进扩展程序页面** → 显示"货叮咚跨境助手已安装成功"。
- **Edge**：同 Chrome（同一安装包），路径为 ⋮ → 扩展 → 管理扩展 → 开「开发人员模式」→ 拖入安装包。
- **360**（不推荐）：插件**不在"扩展程序"里**——点右上角小拼图图标 → 管理 → **高级管理** → 才是 360 的扩展程序页 → 再拖入安装包。
- **自检插件是否装好**：回货叮咚 → `订单` 页有「检测插件」按钮；装好后刷新一下会从"未安装"变"已安装"。订单页「检测插件」右侧也能直接下载安装（不一定要去产品页）。
- **升级提示**：登录时若提示"货叮咚跨境助手浏览器插件已不是最新版"，点"去下载"按教程重装对应插件即可。
- 装不上/下载失败 → 联系货叮咚平台客服。

### 1.3 授权 1688 会员账号（0 元，采购/铺货要用）
> 这是批1**没有的**步骤。后续用 1688 官方 API 采购、1688 精选货源铺货前要先授权 1688 会员账号。

- 位置：`货叮咚 系统 → 授权管理 → 会员账号 → 1688` → 跳转后点「立即登录」。
- **登录方式**：不用另外注册，**用自己的淘宝账号扫码登录**即可。
- 订购：点「立即订购」→ 勾选服务协议 → **付款显示 ¥0（不收费，只是授权账号）** → 提交订单 → 显示"已成功提交订单" → 勾选「授权」。
- 成功标志：自动跳回货叮咚，下方出现 **账号名称 / 授权时间 / 授权平台**，状态「已授权」。

## 2. 采集：先配过滤规则，再采货

### 2.1 采集配置（第 3 步，采集前先设，过滤垃圾货源）
- 位置：`产品 → 采集配置`。作用：**采集时自动过滤掉大部分不合规/不想要的货源**。
- 可设置的过滤维度（采集时自动跳过）：
  - 已采集、已下架、不包邮、带品牌、零库存、预售商品 的货源
  - **客单价过低或过高**的商品
  - 按**标题/描述/品牌关键词**过滤货源
  - 商品视频**是否采集**
- 支持**按模板管理**，可切换不同采集配置；设好记得**保存配置**。

### 2.2 商品采集（第 4 步，多平台）
> ↔批1 §2.2 已提"从源头平台采集、自带视频自动采集"。增量：**支持 1688/拼多多/淘宝多种入口 + 1688 精选货源一键铺货**。

- 入口：`产品 → 插件采集`（需先装好 §1.2 插件）。
- **四种采集方式**：
  1. **1688**：打开 1688 官网商品页 → 点"采集商品" → 完成。
  2. **拼多多**：打开拼多多官网商品页 → 点"采集商品" → 完成。
  3. **链接采集**：粘贴 **1688 或淘宝**链接 → 点采集 → 完成。
  4. **1688 精选货源**（货叮咚提供）：点「1688 精选货源」→ 找到对应货源商品 → 点「一键铺货」完成采集。
- **采集完的商品统一保存到「采集箱」**（后续认领从采集箱选）。

## 3. 认领配置（第 5 步）：把采来的货"标准化"

> 这是货叮咚的核心增值环节，批1只一笔带过"认领"。"认领"=把采集箱里的原始货按一套规则**自动补全/标准化/套价/翻译**，生成可发布的**草稿商品**。

- 位置：`产品 → 认领配置`。作用：自动标准化、补全缺失商品信息、精准匹配平台类目，提高店群发品效率。
- **不同平台建不同认领模板**（这里选 TikTok 模板）；**不同站点可独立配置**。
- 认领模板可配置项：
  - **价格配置**：按公式定价，**模板自动算价格**（实际算价靠 §4 定价模板）。
  - **认领规则 / 库存预设**：采集到无库存时自动填默认值；多店库存规则。
  - **统一 SKU 生成**：按规则生成"前缀 + 属性编码"。
  - **图片**：自动补主图、加水印（"一键搞定"）。
  - **标题/描述优化**：批量加品牌前缀、增删关键词、**过滤手机号**等。
  - **产品翻译**：自动翻译商品文字 + **商品图片翻译（⚠️付费功能，按需开启）**；可选翻译范围/语言/翻译模型。
  - **违禁词/风控**：过滤违规词、敏感信息。
  - **细节配置**：物流、尺码图、包裹重量/尺寸、预售等自动预设。
- 按模板保存管理。
- **认领发布动作**：`采集箱 → 选中商品 → 认领 → 选 TikTok 平台 → 选 TikTok 店铺 → 选「认领并立即发布」或「认领并定时发布」→ 选认领配置模板 → 确定`。认领后会**按模板重新生成一个草稿商品**。

## 4. 定价模板（第 5 步配套，本文最有料）

> ↔批1 §5 已讲过定价模板的**逻辑层**（藏价五项成本、5% 达人佣金、越南不低于5折、马来 VAT 10%/≤500马币 等）。**本节是货叮咚后台"创建模板"的逐字段实操 + 录制时的具体费率示例值**。批1 §5.1~5.2 仍然是定价"思路"权威，本节是"在货叮咚里怎么点、每格填什么"。

### 4.1 定价模板是什么 / 位置
- 位置：`产品 → 定价模板 → 创建模板`。
- 定义：**仅作藏价模板**——把"平台佣金、交易手续费、达人佣金、运费、货代费"等各类成本**藏进售价**，避免按会员价售卖导致亏损。
- ⚠️ 套了定价模板发布到店铺后台后，**仍要在店铺后台做相应折扣促销**（藏价≠最终价）。
- 模板分**四个板块**：①模板信息 ②成本信息 ③利润信息 ④尾数处理。**带星号(\*)=必填，不填存不了。**

### 4.2 ① 模板信息
- **模板名称\***：自己起、方便记忆（教程例：直接写"泰国"）。
- **模板类型**：「公用」/「专属」二选一（点字段左侧小问号看解析）。
- **站点**：选目标国（教程选泰国；**本店选马来西亚**）。
- **店铺类型**：「跨境」/「本土」——**本店选「跨境」**。

### 4.3 ② 成本信息（含具体费率示例 ⚠️均为泰国站录制值）
- 选好"店铺类型"后，系统会**自动带出一套费率**，但教程**建议按平台最新政策手动改**。
- 教程**泰国站手填示例**（⚠️**仅泰国、仅录制当时**，马来/当期都不同，务必去学习中心核）：

| 字段 | 教程示例值（泰国） | 备注 / 可信度 |
|---|---|---|
| 平台佣金费率 | **9.63%** | ⚠️以官方为准；批1曾出现"9.63"同口径 |
| 交易手续费率 | **3.21%** | ⚠️注：店铺后台交易手续费=**折后价 × 3.21%** |
| 提现手续费率 | **1%** | 教程称"一般填1、基本不变" |
| 增值税率 | **0**（泰国此处填0） | ⚠️与批1冲突点见 §8 |
| 进口关税税率 | **32.68%** | ⚠️泰国示例值，存疑偏高，务必核（见 §8） |
| 达人佣金税率 | **5%**（示例） | 见 §4.5 达人佣金逻辑 |
| 电商增长服务费 | **8.03%** | ⚠️以官方为准 |
| 平台基础设施费 | **0.23%** | ⚠️以官方为准 |

- **头程运费**：选「默认」即可。两种来源：①官方跨境物流 ②第三方物流（要自己发到海外仓）。**本店无货源模式 = 货寄到货代 → 货代寄到「关仓」→ 关仓扫描出境，所以选「官方跨境物流」即可。** 右侧有「官方运价表」可点开按站点查头程跨境物流成本。
- **尾程费用（尾程运费）**：系统自动填了示例值 16.45，但**教程建议直接填 0**——尾程是**买家支付的**，卖家不出。该字段是必填项（带星号），填0即可。它"仅用于计算税率和交易手续费"，但实测交易手续费是按折后价×3.21% 算的，所以填0不影响商家运费（商家运费由产品重量算，填0/填16.45 商家运费都不变）。
- **货代费用**：教程口径 **2~3 元/件**，示例直接填 **3**。（与 §7 利润明细里回填的货代费是同一笔成本，那边示例填 2 元/件。）
- **其他费用**：按自己定价需要自定义。
- **重量取值**：建议默认选「商品包裹重量」。

> 找费率政策的官方路径（教程亲授）：`TikTok 店铺后台 → 右上角头像 → 帮助与支持 → 学习中心` → 搜关键词（如"泰国佣金"）。或找飞书群助教要政策网址。**这是本文里唯一权威的数字来源——所有费率以此为准。**

### 4.4 头程 vs 尾程运费怎么算（教程举例）
- 头程运费 = 卖家承担的跨境物流成本，按「官方运价表」对应站点查。
- 尾程运费 = 买家承担，**模板里填0**。
- 算例（泰国、300克）：运价表泰国"起重10克、续重每10克1泰铢"→ 300克寄到泰国买家 = **约 30 泰铢**商家运费（系统按产品重量自动算，不随尾程字段变）。⚠️ 这是泰国运价表口径，**马来要查马来运价表**。

### 4.5 ③ 利润信息
- **利润方式**：三选一（点小问号看解析）：
  - **售价利润率**（教程默认、推荐）——算出的**利润空间较大**。
  - 成本利润率 / 固定利润率——利润空间较小，很少用固定利润率。
- **利润率**：教程填 **20%**。提醒：填太高 → 套模板后店铺本地展示价会偏高。
- **折扣**：填**数字**，且**国内外折扣反着打**：
  - 填 **7** = 店铺后台做 **30% off（七折）**
  - 填 **6** = 店铺后台做 **40% off（六折）**
  - ⚠️ 这与批1 §5.3"6折=让利40%"是**同一回事的两种表述**（货叮咚填"7"对应店铺端"让利30%"）。
  - **建议一定填折扣**——只填利润不填折扣容易亏损（店铺后台要靠模板做折扣）。

### 4.6 达人佣金的藏价逻辑（教程反复强调，重要）
- 模板里"达人佣金税率"藏多少 = 你打算给达人的点数。
- 若**模板藏5% 但店铺后台实际给达人15%** → 多出的 **10% 从你自己利润里扣**。
- 若模板藏5% 且后台实际给5% → 刚好抵扣。
- 若**前期不给达人佣金** → 这藏的5% 就**变成你自己的利润**。
- 算例：模板利润率20% + 达人佣金藏5%，若前期不付达人佣金 → **总利润 = 25%**；若实际付达人15% → 最终利润只剩 **10%**。
- ↔批1 §5.1 同口径（视频带货佣金低3–5个点；深度直播合作可能谈到10–15%，届时单价要上调）。

### 4.7 ④ 尾数处理 + 价格试算 + 保存
- **尾数处理**：默认「不处理」即可，尾数金额非必填、可不填。
- **价格试算**：点「价格试算」，输入采购价 + 重量 → 看到：本地展示价**原价（折前）**、本地展示价**折后价**、扣掉各项费用后**剩下的利润**。算例：采购价20、重量300克 → 按七折试算给出折前/折后价与利润。
- ⚠️ **设好务必点「保存」**，否则模板不保存。保存后在定价模板列表可见。
- 货叮咚另有"图文解析版定价模板"，可找飞书群助教索取。

## 5. 发布：发布配置 → 编辑 → 批量发布（第 6/7/8 步）

### 5.1 发布配置（第 6 步，发布前先设）
- 位置：`产品 → 产品发布`（发布前先设「发布配置」）。
- 可配置：店铺细节、**多国配置**；发布时自动做：**重复检测、站点语言翻译、长度优化**、仓库与多国发布配置。
- **自动默认发货仓库 + 自动关联多国站点发布**。选配置模板 → 保存配置。

### 5.2 产品编辑（第 7 步，认领后再精修草稿）
- 对认领生成的**草稿商品**可二次编辑：
  - 标题、图文描述编辑；**批量管理/编辑/翻译描述图**（选翻译站点语言后保存）。
  - 类目和属性按实际填充。
  - 规格名称、SKU 名称编辑；**批量改 SKU 价格 / 批量改库存 / 平台 SKU 生成 / 批量改重量**。
  - 产品图片优化：批量删不合规图、批量编辑、批量翻译图片语言（SKU 图同流程）。
  - **尺码表**：用图片样式或直接选店铺模板。
  - 支持上传/改产品视频、优化包裹重量尺寸。
  - 可「保存修改」或「保存并发布」。

### 5.3 产品发布（第 8 步）
- 草稿商品可直接「发布」或「定时发布」，支持二次编辑后发布。
- **批量发布**：勾选多个草稿商品 → 批量发布 → **输入间隔时间** → 批量发布。
- 发布状态：定时发布 / 发布中 / 发布成功 → 发布成功后查"在线商品" → **审核成功后才能开启售卖**。

## 6. 采购派单（出单后采货）：官方 API vs 插件拍单

> 两种把 TikTok 订单转成 1688/拼多多采购单的方式。批1未细化此环节。

### 6.1 官方 API 采购（推荐，仅淘宝/1688）
- 位置：`订单 → 官方 API 采购`（**支持淘宝、1688**）。教程演示 1688 批量采购：
  1. 批量选中采购订单 → 选「批量1688采购」→ 进发起采购页面。
  2. **未绑定货源**：可「快捷搜同款」绑定 / 选同款货源快捷绑定 / 一键移除未绑定货源商品。
  3. **不满足批发条件**的商品可切换「代发下单」。
  4. **首次下单选下单规格**，之后自动默认匹配规格（提高派单效率）。
  5. 预览采购单 → **二次核对**：单笔提交金额、运费、商品利润/预估总利润、总采购数量、仓库地址。
  6. 确认下单账号 → 提交订单（可稍后登录支付，或「提交并支付」跳转 1688 批量付款）。
  7. **付款完成自动绑定订单号、同步采购成本**；商家发货后**自动同步发货单号、提交仓库打包**。
- 卖点：全程 1688 官方 API，"更快更稳定"。

### 6.2 插件拍单（支持 1688/淘宝/拼多多）
- 位置：`订单`（货叮咚 ERP 插件拍单，**在线自动同步发货单号**）。
- 前置：上方检测「派单插件是否安装、派单账号是否在线」；对**未绑定货源**的商品先关联货源（手动加电商平台货源链接 / 快捷搜同款关联）。
- 流程：批量选待采购订单 → 「批量采购拍单」→ 核对仓库信息 → 发起采购。
  - **首次先设「智能派单」**：自动匹配 SKU、自动匹配数量、跳过支付利润预警，**开启自动支付后确认保存**。
  - 首次下单手动选派单 SKU、核对商品利润和地址 → 「立即支付」系统自动支付 → 付完自动跳下一个采购订单。
  - 商家发货后**定时同步发货单号、自动提交仓库打包**；全部完成自动绑定采购订单号与采购成本。

## 7. 利润明细：回填采购金额 + 货代费用（批1完全没有的新主题）

> 这是**让利润算准的关键收尾**：采购派单后，把每单的**实际采购价**和**货代费**填回货叮咚利润明细，系统才能算出真实利润。

### 7.1 进入与筛选
- 位置：`货叮咚 数据 → 利润明细`。
- 先选好：**平台**（做 TK 选 TK）、地区、店铺。
- 筛选条件：
  - **订单状态**：勾「已发货」+「已完成」。
  - **结算状态**：勾「已结算」。
  - **订单时间**：自己选好时间范围。
  - 点「查询」。
- 查询后下滑找「**快速过滤**」→ 勾「只显示」→ 把「代发采购」改成「**未录入采购金额**」→ 下方即为待填采购价的订单。

### 7.2 录入采购金额（单个 / 批量）
- **单个录入**：在订单行下方那条横杠**一直往右滑** → 找到「订单成本 → 采购商品成本」→ 在该订单对应格填入采购价（例：填 10 元）→ 点空白处 → 上方提示「成本分摊成功」即完成。
- **批量录入**（同一款商品多单一起填）：
  1. 找到该商品「产品信息 → 商品 ID」→ 鼠标悬停，商品 ID 右边有「复制」按钮，点复制（提示"复制成功"）。
  2. 上滑到「商品 ID」搜索框 → 粘贴 → 点「查询」（筛出同款全部订单）。
  3. 勾选要批量录入的订单 → 「订单成本 → 采购商品成本」→ 点「批量」→ 输入统一采购价（例：15 元）→「修改类型/更新类型」保持**默认不动** → 确认。
- **核对已填**：快速过滤把「已显示」改成「**有录入采购金额**」，下方即已录入的订单。

### 7.3 录入货代费用（单个 / 批量）
- 货代费同样支持单个/批量，口径示例 **2 元/件**：
  - **单个**：在该订单「货代费用」格填 2 → 点空白处 → 提示「分摊成功」。
  - **批量**：勾选订单 → 找「货代费用」→ 点「批量」→ 填 2 元 → 修改/更新类型保持默认 → 确认 → 提示「成本分摊成功」，右侧货代费列显示"2 元/件"。

> 注意：货代费在两处都出现——**定价模板**里藏价时填（§4.3，2~3元/件），**利润明细**里按实际回填（这里，示例2元/件）。两者要对齐，否则模板藏的成本和实际成本对不上。

## 8. 冲突 / 待核实 / 避坑

### 8.1 与批1及官方的数字冲突（务必以官方学习中心为准）
- ⚠️ **增值税率**：本教程泰国模板里"增值税率"填 **0**；但批1 §5.2 明确**泰国 VAT 7%（对所有商品征收）**。原因可能是货叮咚把"进口关税"和"增值税"拆成两格、或税由平台代缴不进藏价——**口径存疑，泰国/马来都要按官方核**。本店做**马来**：批1记的是 **VAT 10%、适用售价≤500 马币**，模板里该怎么填以学习中心马来政策为准。
- ⚠️ **进口关税 32.68%**：泰国示例值，明显偏高，疑为录制随手填的演示数，**不可照抄**，按当期官方运价/税则核。
- ⚠️ **平台佣金 9.63% / 交易手续费 3.21% / 电商增长服务费 8.03% / 基建费 0.23%**：均为**泰国**录制值。马来的费率不同，**逐项去学习中心查马来站**。
- ⚠️ **折扣方向**：货叮咚填"7"=店铺端"30% off / 让利30%"；批1 §5.3 用"让利40%=6折"表述。换算关系一致（填N → 让利 (10−N)×10%），但**两处填法不同别填混**。

### 8.2 工具性质提醒
- ⚠️ 货叮咚=**第三方 ERP，非官方**。插件、官方 API 采购、收费项随服务商版本变；遇问题联系货叮咚客服/飞书群助教。
- ⚠️ **付费功能**：商品**图片翻译 / 图片语言翻译**是付费功能（§3 认领配置、§5.2 编辑都涉及），按需开启；文字翻译一般免费。↔批1 §2.2 同口径（"图片翻译需另付费"）。
- ⚠️ **360 浏览器不要用**（教程明示下载易带病毒）；优先 Chrome。

### 8.3 合规/铺货避坑（沿用批1，仍适用）
- 本套是**无货源/铺货式**打法（1688/拼多多采集），属灰区，批1消化日志已声明"勿照做"性质；采集配置的"过滤品牌/关键词"务必开，避免仿品牌、标题含品牌词/年份/禁卖品名。
- ↔批1 §2.3 批量上架避坑清单仍适用：**编辑好务必先套定价模板再发布**、服装类目必传尺码图、重量要准（直接影响藏价利润）、带星号必填。
- 上架禁售/限售红线见批1 `商品发布上架与定价.md` §9（含马来专属：矫正型隐形眼镜/框架眼镜禁售、宗教种族歧视材料禁售）。
