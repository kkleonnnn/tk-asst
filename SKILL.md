# TikTok Shop 东南亚运营助理

## 这个 skill 是什么
一份给「自用」的 TikTok Shop 东南亚运营手册 + 操作流程库。它指导 Claude 协助完成开店、选品上架、日常运营与发货、数据分析。它不是一个被训练的模型，而是一份持续整理、可版本控制的知识与流程文档。

## 何时使用
- 需要东南亚 TikTok Shop 的开店流程指导或材料检查
- 需要批量/重复地上下架商品、处理订单发货
- 需要 listing 文案、标题、客服话术、活动规划等运营建议
- 需要解读店铺后台数据、出分析报告

## 知识如何组织
- `reference/` —— 单一知识源，按 9 大板块分：
  `01-开店入驻` / `02-选品` / `03-商品上架` / `04-履约发货` /
  `05-流量增长`(短视频·直播·商城自然流量·付费广告·达人联盟·内容文案) /
  `06-活动大促` / `07-客服与店铺健康` / `08-数据分析` / `09-风控合规`
- `sop/` —— 可执行的标准操作流程（上下架、发货等的分步清单）
- `templates/` —— 模板（listing 文案、报告、客服话术）
- `inbox/` —— 收件箱：新资料先丢这里（大文件只留本地不入库）
- `通用知识包.md` —— 由 reference/ 生成的可移植版本，供其他 AI/人使用（见 README.md）

## 消化规则（两遍消化制）
- **第一遍消化**：任何人用任何模型，把 `inbox/` 资料先消化成结构化 `.md`（仍放 `inbox/`）——**不限模型**。
- **最终整合（写入 `reference/`）= kk 的质量关口**：把 inbox 的 `.md` 校对/去重/核实/规范化后并入 `reference/`，建议用 Opus 4.8+ 把关。**其他人/其他模型不要直接改 `reference/`**（只交第一遍 `.md` 到 inbox）。
- 写入 `reference/` 的改动**走 PR、合并 main 才算正式生效**；并在 `reference/_消化日志.md` 记一笔。
- 整体协作机制与角色分工见 `README.md` 与 `sop/角色分工.md`。

## 使用原则（重要，每次执行前都要遵守）
1. **人在环**：任何改动真实店铺的操作（改价、上下架、发货、回复买家）都必须经人工确认后再执行。
2. **不碰敏感信息**：账号密码、API key、买家隐私数据绝不写进本仓库。
3. **风控意识**：自动化操作保持人类节奏、分批小量，早期不无人值守，避免触发 TikTok 风控。
4. **站点差异**：东南亚各国规则不同，执行任何操作前先确认针对哪个国家站点。
